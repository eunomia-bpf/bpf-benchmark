## Analysis

- Across all 496 available objects, `ROTATE` is the most common family by raw site count (2687 sites), while `WIDE` touches the widest set of programs (62 objects).
- The current on-disk corpus snapshot contains 429 real-program objects. This is smaller than the `162 paired instances (36 unique)` target called out in `docs/kernel-jit-optimization-plan.md` §6.3, so these numbers should be treated as a snapshot of currently available workloads rather than the final paper corpus.
- The densest overall objects are `corpus/build/linux-selftests/tools/testing/selftests/bpf/progs/test_verif_scale2.bpf.o` (992 sites), `corpus/build/linux-selftests/tools/testing/selftests/bpf/progs/core_kern.bpf.o` (818 sites), and `micro/programs/mega_basic_block_2048.bpf.o` (520 sites).
- Within the real-program corpus, the highest-density objects are `corpus/build/linux-selftests/tools/testing/selftests/bpf/progs/test_verif_scale2.bpf.o` (992 sites), `corpus/build/linux-selftests/tools/testing/selftests/bpf/progs/core_kern.bpf.o` (818 sites), and `corpus/build/bcc/libbpf-tools/biostacks.bpf.o` (15 sites).
- The overall site mix is heavily skewed by synthetic micro workloads: micro objects contribute 1074/3073 total sites, and nearly all `ROTATE` opportunities come from that set (877 of 2687).
- In the current real-program corpus, `CMOV` is the only directive family with meaningful coverage (183 sites across 10 objects). `WIDE` appears only in `corpus/bcf/collected/xdp_synproxy_kern.bpf.o` (6 sites), while `ROTATE` and `LEA` have zero hits in this snapshot.
- The §6.3 policy-sensitivity set already contains directive-bearing workloads in this raw-ELF scan: `packet_rss_hash`, `local_call_fanout`, `branch_fanout_32`, `mega_basic_block_2048`.
- For paper positioning, the strongest claim this census supports is breadth of candidate coverage, not acceptance. The raw ELF scan found sites in 57/429 real-program objects; verifier-inserted instructions and exact validator checks still need an xlated-program acceptance pass to complete the §6.3 story.
- For evaluation claims, this means the micro suite already exercises all four families, but the current real-program snapshot does not yet support a strong claim that all four families are common in deployed workloads. That gap should be closed either by expanding the real corpus or by narrowing the claim to the families that actually appear.

## Methodology Notes

- This census scans raw instructions from every executable ELF section in each `.bpf.o` file. It does not require loading the program into a kernel or VM.
- Pattern matching is a direct Python translation of `scanner/src/scanner.cpp` and `micro/runner/src/kernel_runner.cpp`, including the rotate variants present in the implementation.
- Counts are candidate sites, not accepted rules. They are therefore appropriate for a coverage census, but not a substitute for xlated-program acceptance-rate measurements.
