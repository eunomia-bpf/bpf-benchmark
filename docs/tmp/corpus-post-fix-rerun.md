## Context

- **Previous corpus result (v2 fixed, 2026-03-12)**: geomean 0.875x — declared **invalid** by #162
- **Root cause of invalidity**: `corpus_dummy_packet_64.bin` was a 64-byte zero/random buffer,
  not a valid Ethernet/IPv4/TCP frame. XDP/TC programs drop on early parse failure, so
  the eBPF execution bails out before reaching any recompile sites.
- **Fix applied**: `micro/orchestrator/corpus.py::_build_corpus_packet()` now generates a valid
  64-byte IPv4+TCP SYN packet (dst_mac=broadcast, src=10.0.0.1, dst=10.0.0.2, dst_port=80, SYN flag,
  correct IPv4 checksum). Verified by `xxd`: `45 00 00 32 ... 26c4 ...`

## What This Rerun Tests

- **bzImage**: build #38, commit `daca445b1` (BEXTR fix: removed `dst_reg == src_reg` guard)
- **Policy mode**: `--use-policy` (v3 policy files from `corpus/policies/`, auto-generated by
  `generate_default_policies.py`, skip CMOV, apply rotate/lea/extract/endian/branch-flip)
- **Targets**: 166 packet-runnable programs (from `docs/tmp/corpus-runnability-results.json`)
- **Repeat**: 200 per program

## Expected Improvements Over v2

1. **Packet validity**: Programs that previously exited early due to invalid packet now run to completion
2. **BEXTR fix**: `extract` family sites now correctly optimize 3-insn→2-insn patterns
3. **CMOV skip**: Policy files already exclude CMOV (known net-negative)

## Notes on Policy Coverage

The `corpus/policies/` directory has v3 policy files generated by `generate_default_policies.py`.
These were generated against the inventory scan (old bzImage), so:
- CMOV sites are correctly excluded
