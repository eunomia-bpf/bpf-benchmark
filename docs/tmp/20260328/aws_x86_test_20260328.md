# AWS x86 test report (2026-03-28)

## Result

Status: partial validation only.

- AWS credentials/profile were usable.
- Local `daemon` / `runner` build and local `smoke` sanity check passed.
- `runner/scripts/aws_x86.sh` passed `bash -n`.
- `make aws-x86-full` successfully launched an x86 EC2 instance and reached SSH/setup.
- I did **not** allow the remote timing benchmark to proceed, because other repo performance jobs were already running and `AGENTS.md` forbids running another timing benchmark in parallel.
- I interrupted `make aws-x86-full` during setup. In this interrupted, piped invocation, the instance was **not** auto-terminated by the observed cleanup path; I had to run `make aws-x86-terminate` manually.
- Final AWS state is clean: the x86 instance is terminated, `.cache/aws-x86/state/instance.env` is absent, and `.cache/aws-x86/results/` is empty.

This means the new x86 AWS flow is only validated up to `launch + SSH + remote prereq install + start of local kernel-module staging`. It is **not yet validated end-to-end** through `setup verify + benchmark + automatic cleanup`.
