- add `libtinfo.so*|libncurses.so*|libncursesw.so*` to runtime lib staging in `runner/scripts/cross-arm64-build.sh`

### Attempt 2: stock `make cross-arm64` after ncurses fix

Observed behavior:

- ARM64 docker image rebuilt successfully
- ARM64 `micro_exec` runner build progressed under `qemu-binfmt`
- daemon step then failed immediately

Failure:

```text
CMake Error: The source directory "/workspace/daemon" does not appear to contain CMakeLists.txt.
```

Interpretation:

- current repo `daemon/` is a Rust crate with `Cargo.toml`, not a CMake project
- the existing cross-build script is stale for the current daemon layout

Fix applied:

- add `cargo` and `rust` to `runner/docker/arm64-crossbuild.Dockerfile`
- change daemon build in `runner/scripts/cross-arm64-build.sh` from `cmake ...` to:

```bash
cargo build --release --manifest-path /workspace/daemon/Cargo.toml
```

## 9. Overall Verdict

Short answer:

- BpfReJIT on ARM64 is partially working, but not fully healthy.

What is confirmed working:

- ARM64 kernel boots under QEMU
- ARM64 BPF JIT is enabled
- ARM64 `BPF_PROG_REJIT` core path works (`rejit_poc` 6/6 pass)
- ARM64 kinsn discovery works after rebuilding the current ARM64 test binary
- rotate/select/endian/bulk memcpy/LDP/STP/extract ARM64 smoke paths all pass in at least one current-source test run
- ARM64 kinsn modules build and load successfully (`6/6`)

What is not working:

- `bulk_memset_reg_apply` currently fails with `EINVAL`
- `bulk_memset_max_len_zero_fill` reproducibly crashes the ARM64 guest kernel in `bpf_stack_liveness_free`
- `cross-arm64` infra is stale and needed fixes; final green status was not reached within this run

Final verdict:

- it is not accurate to claim "ARM64 BpfReJIT fully works" yet
- it is accurate to claim:
  - ARM64 support is substantially functional
  - core REJIT and many ARM64 kinsn paths work in QEMU
  - there is at least one real ARM64 correctness bug left in bulk-memory memset handling

## 10. Problems Encountered and Solutions

1. Stale ARM64 kernel Git worktree registration.
   Solution: `git -C vendor/linux-framework worktree prune`

2. `cross-arm64` runner image missing ncurses/terminfo build/runtime deps.
   Solution: add `ncurses-devel`; copy `libtinfo/libncurses` into runtime bundle

3. `cross-arm64` daemon build script assumes a CMake daemon project, but current repo daemon is Cargo-based.
   Solution: switch script to `cargo build --release`

4. Prebuilt `tests/unittest/build-arm64/rejit_kinsn` was stale relative to source.
   Solution: rebuild minimal ARM64 unittest binaries with `aarch64-linux-gnu-gcc`

5. Current ARM64 bulk-memory memset zero-fill path triggers kernel Oops.
   Solution in this run: no fix yet; bug reproduced and isolated
