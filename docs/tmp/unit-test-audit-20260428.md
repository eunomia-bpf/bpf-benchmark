## Standard Applied

Reasonable unit tests cover logic branches, state changes, calculations, conversions, boundaries, error paths, external ABI/layout/serialization contracts, or bug regressions. ABI/layout tests must verify field offsets or encoded format, not just `size_of`.

Deleted tests matched one or more low-value categories: trivial getter/default/data-struct assertions, upstream/std behavior checks, self-equality tautologies, mock-only tests, implementation-detail order smoke, size-only layout checks, test-helper self-tests, host-dependent smoke with no clear expected result, duplicate coverage, or no-op tests.

## Guideline Added

- `CLAUDE.md`: added `### Unit Test Quality` under `## Design Rules`.
- `docs/kernel-jit-optimization-plan.md`: added a Chinese unit-test quality rule in the top editing rules and referenced `CLAUDE.md`.

Guideline content:
- Do not add unit tests without a clear bug-detection purpose.
- Good tests cover logic branches, state changes, calculations/conversions, boundaries, error paths, external ABI/layout/serialization contracts, or bug regressions.
- ABI/layout tests must verify field offsets or encoded format, not only `size_of`.
- Do not test trivial getters/setters, std/upstream behavior, self-equality tautologies, mock-only behavior, readability examples, pure const aliases, or duplicate coverage.
- Before adding a test, answer: what specific bug would this failure identify?
