## Scanner / Recompile Blocker

The final attempt built the scanner successfully, but the remote pin-and-enumerate path failed before enumeration started.

For `tracee`, `tetragon`, and `katran`, `bpftool prog loadall` returned:

```text
Error: expected no more arguments, 'type', 'map' or 'dev', got: 'kernel_btf'?
```

Observed effect:

- `loadall.rc=255` for all three corpus cases
- pin directory paths were created under `/sys/fs/bpf/arm64-e2e-smoke-*`, but program pinning did not complete
- no `enumerate*`, `prog_info*`, or `recompile*` artifacts were produced in the final result bundle
- because enumerate never ran, forms found and sites applied are unavailable for this smoke
- because the pinned-program path was blocked, `BPF_PROG_JIT_RECOMPILE` was not exercised remotely in this run
